Building on Srini's explanations in [Understanding HTTP](https://www.youtube.com/watch?v=a3C1DMswClQ), here are practical Go (Golang) examples showing **when** and **how** to set these headers in a real-world backend environment.

### **1. Setting Security & Identity Headers**
**Scenario:** You want to ensure your API only communicates over HTTPS and identifies itself to the client.

```go
func securityMiddleware(next http.Handler) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        // HSTS: Forces the browser to use HTTPS for the next year [00:13:22]
        w.Header().Set("Strict-Transport-Security", "max-age=31536000; includeSubDomains")
        
        // Prevent Clickjacking [00:13:46]
        w.Header().Set("X-Frame-Options", "DENY")
        
        // Identify your server version or type (optional)
        w.Header().Set("Server", "Go-Backend-Service")

        next.ServeHTTP(w, r)
    })
}
```

### **2. Content Negotiation (Multi-format Support)**
**Scenario:** Your server supports both JSON and XML. You check the client's `Accept` header to decide what to send.

```go
func handleFlexibleData(w http.ResponseWriter, r *http.Request) {
    acceptHeader := r.Header.Get("Accept") // [01:03:26]

    if strings.Contains(acceptHeader, "application/xml") {
        w.Header().Set("Content-Type", "application/xml") // [01:04:14]
        w.Write([]byte("<user><name>Srini</name></user>"))
    } else {
        // Default to JSON
        w.Header().Set("Content-Type", "application/json")
        w.Write([]byte(`{"name": "Srini"}`))
    }
}
```

### **3. Managing the Cache Lifecycle**
**Scenario:** You have a "Roadmap" resource that changes rarely. You use `ETag` and `Cache-Control` to save bandwidth.

```go
func roadmapHandler(w http.ResponseWriter, r *http.Request) {
    roadmapData := "Backend First Principles Roadmap"
    // In a real app, generate this hash from the database content [00:57:05]
    currentEtag := `"v1.0.5-hash"` 

    // 1. Check if the Client's version is still fresh [00:58:17]
    if r.Header.Get("If-None-Match") == currentEtag {
        w.WriteHeader(http.StatusNotModified) // 304 [00:59:15]
        return
    }

    // 2. Set headers for the new version
    w.Header().Set("ETag", currentEtag)
    w.Header().Set("Cache-Control", "public, max-age=3600") // Cache for 1 hour [00:56:47]
    w.Write([]byte(roadmapData))
}
```

### **4. Handling Authentication (Authorization)**
**Scenario:** Protecting a route. The client must send a token, or the server rejects it.

```go
func protectedHandler(w http.ResponseWriter, r *http.Request) {
    authHeader := r.Header.Get("Authorization") // [00:15:57]

    if authHeader == "" || !strings.HasPrefix(authHeader, "Bearer ") {
        // Return 401 if token is missing [00:46:47]
        w.WriteHeader(http.StatusUnauthorized)
        w.Write([]byte("Missing or invalid token"))
        return
    }

    w.Write([]byte("Access granted to sensitive data"))
}
```

### **5. Compression (Reducing Payload Size)**
**Scenario:** You are sending a massive list of entries. You check if the client supports `gzip`.

```go
func largeDataHandler(w http.ResponseWriter, r *http.Request) {
    // Check if client supports gzip [01:07:49]
    if strings.Contains(r.Header.Get("Accept-Encoding"), "gzip") {
        w.Header().Set("Content-Encoding", "gzip") // [01:08:46]
        gz := gzip.NewWriter(w)
        defer gz.Close()
        
        // Write large data to the gzip writer
        gz.Write([]byte("...huge amount of data..."))
    } else {
        w.Write([]byte("...huge amount of data..."))
    }
}
```
