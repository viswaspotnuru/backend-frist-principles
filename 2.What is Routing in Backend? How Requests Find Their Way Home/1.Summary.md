This video explains **Routing in Backend**, describing it as the "where" of a request, whereas HTTP methods describe the "what" (the intent). Routing involves mapping URL parameters to server-side logic [01:55](http://www.youtube.com/watch?v=SubuU1iOC2s&t=115).


### **1. Static vs. Dynamic Routes [04:36](http://www.youtube.com/watch?v=SubuU1iOC2s&t=276)**
* **Static Routes:** These are constant URL paths like `/api/books` that do not change and always return a fixed type of response [04:45](http://www.youtube.com/watch?v=SubuU1iOC2s&t=285).
* **Dynamic Routes:** These include variable parameters, often marked with a colon (e.g., `/api/users/:id`). The server extracts these **Path Parameters** (also called Route Parameters) to perform specific actions, like fetching a unique user by their ID [05:32](http://www.youtube.com/watch?v=SubuU1iOC2s&t=332).

### **2. Path Parameters vs. Query Parameters [08:50](http://www.youtube.com/watch?v=SubuU1iOC2s&t=530)**
* **Path Parameters:** Part of the URL itself (e.g., `/users/123`). They are used for **semantic expression**, identifying a specific resource [11:06](http://www.youtube.com/watch?v=SubuU1iOC2s&t=666).
* **Query Parameters:** Key-value pairs following a question mark (e.g., `/search?query=book`). They are typically used with `GET` requests for metadata like **filtering**, **sorting**, or **pagination** (e.g., `?page=2&limit=20`) [12:07](http://www.youtube.com/watch?v=SubuU1iOC2s&t=727).

### **3. Nested Routes [15:09](http://www.youtube.com/watch?v=SubuU1iOC2s&t=909)**
Nesting expresses relationships between resources. For example, `/api/users/123/posts/456` semantically means "fetch post 456 belonging to user 123" [18:19](http://www.youtube.com/watch?v=SubuU1iOC2s&t=1099).

### **4. Route Versioning & Deprecation [18:54](http://www.youtube.com/watch?v=SubuU1iOC2s&t=1134)**
* **Versioning:** Using prefixes like `/v1/` or `/v2/` allows developers to introduce **breaking changes** (e.g., changing response structure) without breaking existing client apps [20:24](http://www.youtube.com/watch?v=SubuU1iOC2s&t=1224).
* **Deprecation:** Versioning provides a "window" for frontend engineers to migrate to the new format before the old version is eventually removed [21:29](http://www.youtube.com/watch?v=SubuU1iOC2s&t=1289).

### **5. Catch-all Routes [22:23](http://www.youtube.com/watch?v=SubuU1iOC2s&t=1343)**
A catch-all route (often using a wildcard like `*`) is defined at the end of the routing logic. It captures any requests to endpoints that the server does not explicitly handle, allowing it to send a user-friendly "Route Not Found" message [23:16](http://www.youtube.com/watch?v=SubuU1iOC2s&t=1396).
